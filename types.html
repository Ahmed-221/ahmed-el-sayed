Types of operating systems

Single-user single-task
The simplest types of operating systems serve one user at a time, which is single-tasking, in other words, and it can implement only one program at a time. Examples include the DOS-MS system.
Single user multi-tasking
This system has the ability to run more than one program simultaneously, as the central processing unit (CPU) moves between missions very quickly. Examples include: Windows95, Windows98.
Multiplayer single user task
This system allows a number of people to implement one program at a time. And each user is provided with an input and output station that communicates with the central computer and this organization is called a time sharing system (Timesharing) because the operating system orders the computer to move very quickly between users after giving them fixed periods of time to use the CPU and these periods are called Time Slices. Too short (about a thousandths of a thousandth of a second) so that every user fancies that he has the entire computer's attention, for example Windows NT.
For multi-user multi-tasking
It is a modern operating system that allows the user to run multiple programs even if the multiprocessor (CPU) is not sufficient for that. Operating systems distribute processor time between these programs so that each program takes a specific time from the processor and then pauses the program and gives another program this time. This means that during one moment, one program running on the processor and operating system is changing very quickly as parts of a second. The process of allocating processor time is called scheduling, where the operating system maintains a list of programs that the user has run, and the scheduling process distributes time for each program in this list to benefit from the processor at this time

Operating systems are used on most, but not all, computers.

The simpler computers with smaller implanted systems and many early computers without an operating system, instead of operating systems, have relied on application programs to manage hardware with their knowledge, perhaps with the help of libraries designed for this purpose.
